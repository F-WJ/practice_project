  • git（wsl）
    • 安装（wsl已经帮你安装）
      • sudo apt install git
    • 配置
      • 配置文件存放位置
        • /etc/gitconfig 文件: 包含系统上每一个用户及他们仓库的通用配置。 如果使用带有 --system 选项的 git config 时，它会从此文件读写配置变量。
        • ~/.gitconfig 或 ~/.config/git/config 文件：只针对当前用户。 可以传递 --global 选项让 Git 读写此文件。
        • 当前使用仓库的 Git 目录中的 config 文件（就是 .git/config）：针对该仓库。
        • 每一个级别覆盖上一级别的配置，所以 .git/config 的配置变量会覆盖 /etc/gitconfig 中的配置变量。
        • 在 Windows 系统中，Git 会查找 $HOME 目录下（一般情况下是 C:\Users\$USER）的 .gitconfig 文件。 Git 同样也会寻找 /etc/gitconfig 文件，但只限于 MSys 的根目录下，即安装 Git 时所选的目标位置。
      • 初始化
        • SSH公钥
          • 运行$ ssh-keygen命令， 在密钥的存储位置（默认是 .ssh/id_rsa）找到id_rsa.pub密钥复制到github上。
        • 用户信息（--global是全局）
          • $ git config --global user.name "name"
          • $ git config --global user.email name@example.com
    • git加速（ssr，速度看情况，有时候加速也很慢）
      • git config --global http.proxy 'socks5://127.0.0.1:1080'
      • git config --global https.proxy 'socks5://127.0.0.1:1080'
  • 使用ss加速（wsl）
    • window安装ssr
    • 安装genpac工具
      • pip install genpac
    • 生成配置文件
      • genpac --proxy="SOCKS5 127.0.0.1:1080" -o autoproxy.pac --gfwlist-url="https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt"
    • 编辑/etc/profile文件（端口和ssr客户端保持一致）
      • export http_proxy=http://127.0.0.1:1080
      • export https_proxy=http://127.0.0.1:1080
      • export ftp_proxy=http://127.0.0.1:1080
    • 重启配置（让修改后配置生效）
      • source /etc/profile

        • Install python and pyinstaller on Windows

    • vscode支持wsl python讨论（关注中）

    • 现今方案（python，git可以无缝对接，但跳转定义、调试等内容没法使用，等待vscode-python插件完善）
      • window和wsl路径转换（注意，vscode无法识别anaconda版本）
        • 环境配置（wsl）
          • 安装一些必备工具：sudo apt install -y make build-essential libssl-dev libffi-dev python3-dev（不安装也可以，没影响）
          • 安装 Python 3.7（或者你想要的 Python 版本）：sudo apt install python3.7
          • 安装 Python 包管理 pip：sudo apt install python3-pip
        • python插件（wsl）
          • 安装 pylint：pip3 install pylint（实时代码风格检查工具）
          • 安装 exuberant ctags：sudo apt install exuberant-ctags （快速定位工具）
          • 安装 autopep8：pip3 install autopep8 （代码美化插件 ）
        • vscode配置文件（window）
          • {
          • "terminal.integrated.shell.windows": "C:\\Windows\\System32\\wsl.exe",
          • "git.path": "D:\\code\\vscodetools\\wslgit.exe",
          • "code-runner.runInTerminal": true,
          • "code-runner.terminalRoot": "/mnt/",
          • "python.pythonPath": "D:\\code\\vscodetools\\wsl\\python3.exe",
          • "python.linting.pylintPath": "D:\\code\\vscodetools\\wsl\\pylint.exe",
          • "python.workspaceSymbols.ctagsPath": "D:\\code\\vscodetools\\wsl\\ctags.exe",
          • "python.formatting.autopep8Path": "D:\\code\\vscodetools\\wsl\\autopep8.exe"
          • }

      • wslgit
        • 简介
          • wslgit 提供了一个小巧的转换工具，让 Linux 侧的 git 能够直接被 Visual Studio Code 版本控制工具识别到。
        • 下载
          • https://github.com/andy-5/wslgit/releases
        • vscode setting设置
          • "git.path": "C:\\$更换为 wslgit.exe 的路径$\\wslgit.exe"
        • 提升 Git on WSL 的性能
      • Code Runner
        • 简介
          • Code Runner 是一个能够一键执行编译运行的 VSCode 插件，支持多达 44 种语言。
          • 支持在代码工作区右键选择 Run Code、在 Command Palette (Ctrl + Shift + P) 输入 Run Code、或者直接快捷键 Ctrl + Alt + N 快速编译执行代码。
        • 下载
          • 在vscode商店下载
        • 设置（setting）
          • "code-runner.runInTerminal": true
          • "code-runner.terminalRoot": "/mnt/"
